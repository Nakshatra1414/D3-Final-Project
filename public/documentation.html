Online Shopper Behavior Analysis - Documentation

1. Project Overview

The goal of this project was to design and implement an interactive data visualization using D3.js that enables meaningful exploration of a real-world dataset. The visualization analyzes online shopper behavior based on visitor type, time period, and browsing behavior, emphasizing the full visualization pipeline: data loading, cleaning, transformation, visual encoding, and interaction design.

2. Dataset Description

Source: Kaggle - Online Shoppers Purchasing Intention Dataset

The dataset contains session-level e-commerce data including numerical and categorical attributes. Key attributes used in the visualization:

  • Month – Month of the visit
  • VisitorType – New Visitor, Returning Visitor, or Other
  • Weekend – Whether the visit occurred on a weekend
  • Revenue – Whether the session resulted in a purchase
  • ProductRelated – Number of product-related pages viewed
  • PageValues – Estimated value of visited pages
  • BounceRates
  • ExitRates
  • TrafficType

3. Design Process

Initial sketches explored bar charts, line charts, and flow diagrams. Questions guiding the design included:

  • How does behavior differ between new and returning visitors?
  • Do weekends influence purchasing behavior?
  • Is there a relationship between product exploration and page value?
  • How do different user paths lead to revenue or no revenue?

The final design uses a multi-view approach:

  • Sankey diagram – to represent user flow
  • Scatterplot – to analyze behavioral relationships

4. Final Visualization Design

A. Sankey Diagram – User Flow Visualization

Purpose: Visualizes how users move through stages: Visitor Type → Month → Weekend/Weekday → Revenue Outcome

Design Choices:

  • Nodes represent categorical stages
  • Link width represents the number of sessions
  • Consistent color palette for readability
  • Clear text labels for all nodes

B. Scatterplot – Page Values vs Product-Related Pages

Purpose: Explores the relationship between ProductRelated pages (x-axis) and PageValues (y-axis), with color encoding for Revenue vs No Revenue.

  • X-axis: Product-Related Pages Viewed
  • Y-axis: Page Value
  • Color encoding: Green → Revenue, Red → No Revenue
  • Semi-transparent points to reduce overplotting

5. Interactivity & Filters

Three filters dynamically update both visualizations:

  • Month
  • Visitor Type
  • Weekend

Filters allow users to isolate specific behaviors (e.g., weekend returning visitors) and encourage exploration and hypothesis testing.

6. Findings & Insights

  • Returning visitors dominate traffic but most do not purchase.
  • Weekend sessions show higher revenue proportion.
  • PageValues strongly correlates with revenue.
  • Many users browse extensively without purchasing, indicating potential usability or pricing barriers.

7. Technical Implementation

  • Built entirely using D3.js (v7)
  • Data loaded and parsed using d3.csv()
  • Scales and axes implemented using D3 modules
  • Sankey layout implemented using d3-sankey
  • Modular JavaScript functions for maintainability
  • Compatible with Google Chrome
